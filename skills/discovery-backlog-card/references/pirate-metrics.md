# Pirate Metrics (AARRR)

A frame for the discovery card's **Success metrics** — it forces the question *"what impact would
this actually have?"* across the customer lifecycle, instead of a vague "we'll measure engagement".

Five stages, three columns. For each stage capture:

- **Goal** — the desired outcome at this stage (what we want to happen).
- **Signal** — the early behavioural indicator that it's happening (a leading sign).
- **Metric** — the quantified measure that proves it (the number we'd track).

| Stage | What it covers | Example signal → metric |
|---|---|---|
| **Acquisition** | Getting the right people to the product / feature. | Visits to the new flow → sign-up / activation-start rate |
| **Adoption** | First successful use — the "aha" (a.k.a. activation). | Completed the core action once → % of new users reaching first value |
| **Retention** *(churn reduction)* | People keep coming back / stop leaving. | Repeat use week-over-week → 30/90-day retention; churn rate |
| **Referral** | Users bring in other users. | Shares / invites sent → invite conversion; referral coefficient |
| **Revenue** | The work moves money. | Upgrades / paid actions → ARPU, conversion to paid, revenue retained |

## How to use it on the card

- **Pick only the stages this opportunity moves.** Most discovery cards move one or two (often
  Adoption and/or Retention). Mark the rest **`n/a this phase`** — an honest, focused table beats a
  padded one.
- **One headline per stage.** A single Goal · Signal · Metric per relevant stage. The card is a
  snapshot.
- **No targets yet.** Specific target numbers, baselines, and instrumentation belong in the
  **brief**, not the discovery card — at discovery stage we're naming *what* we'd measure, not
  committing to a number.
- **Signals before metrics.** If a real metric isn't available yet, a good signal still makes the
  intended impact legible and is itself a discovery question (how would we instrument it?).

## Alternatives

The Vision Canvas references **STDC** and the **HEART framework** as other lenses for success
metrics. AARRR is the default here because its stages map cleanly to "what impact would this have?".
If a HEART/STDC view fits an opportunity better, use it — but keep the card just as short.
