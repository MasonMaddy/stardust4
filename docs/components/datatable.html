Component

Datatable

Beta v1.0.0 Updated 2026-07-07

A tabular data surface for lists of records — children, enrolments, invoices, staff. It carries value-typed sorting (dates and money sort by value, not display text), row selection with a true indeterminate select-all, density and zebra options, loading / empty / error states, a sticky header, visible row-action kebabs, and a pagination footer that never scrolls away. A toolbar layer adds search, filter chips and a filter modal; the same data collapses to cards on phones and handles 10+ columns with the pinned-column Scroll+ pattern. It is the single replacement for the legacy Datatable / Spreadsheet family.

(d) Wide tables — Scroll+ with pinned columns

Office tables routinely carry 10+ columns. Scroll+ is horizontal scroll done properly: the child name and row actions stay pinned (with an elevation shadow once the body scrolls), the scrollbar is always visible, the 44px buttons move columns per click, and edge fades signal the cut-off. Focus the table and use the left/right arrow keys, or scroll it.

4Usage Guidelines

✓ Do
  • Use for lists of records where rows are scannable and comparable — children, invoices, staff
  • Sort by value with typed columns (dates and money never sort as strings)
  • Show a distinct empty state for "no data" vs "no matches" so filters aren't mistaken for missing data
  • Turn every active filter into a removable chip, and keep a live result count
  • Reach for Scroll+ (pinned columns) before hiding data when a table genuinely needs 10+ columns
  • Let --responsive collapse to cards on phones instead of forcing a sideways drag
✗ Don't
  • Hide primary actions behind a right-click — use the visible kebab row-actions
  • Build spreadsheet inline-editing on top of this — that's a separate documented pattern, not shipped
  • Put the pagination inside the scroll region where it can scroll away
  • Crush 12 columns to fit — either pin + scroll, collapse, or curate the columns
  • Use a bare overflow scroll with an invisible scrollbar and no pinned anchor column
  • Re-declare any ds-* rule on the page — link the shared CSS instead

Acceptance criteria

  • Typed sort: text/date/number/status sort by value; header cycles none → asc → desc with aria-sort
  • Indeterminate select-all uses the native property; row selection tints __row--selected
  • Density, zebra and sticky header are independent modifiers; compact keeps interactive cells at 44px
  • Loading (skeleton + live region), empty (distinct block) and error (ds-message-box) states
  • Row-action kebab menu portals to a fixed layer, flips near the viewport bottom, closes on scroll/resize/outside/Escape
  • Pagination footer sits outside the scroll region
  • Toolbar filters produce removable chips + a live count; the filter modal is a shipped ds-dialog
  • Scroll+: pinned select/primary/actions columns with the z-index ladder (1 < 2 < 3), always-visible scrollbar, 44px buttons, edge fades, arrow-key scroll
  • --responsive collapses to cards ≤640px with selection + actions intact
  • prefers-reduced-motion: transitions/pulse/smooth-scroll all disabled
