Component

Tooltip

WIP v1.0.0 Updated 2026-07-07

A small, plain-text label that describes its trigger on demand. Flat dark bubble on an inverse surface with a pointing arrow. It reveals on hover (with an intent delay), immediately on keyboard focus, and on press-and-hold for touch — the legacy tooltip was hover-only, which failed on iPads and for keyboard users. Plain text only; rich or interactive content belongs in a Popover.

ℹ️
Auto-flip — placement is a preferred side. When the preferred side would clip its container, the runtime measures and flips to the opposite side (the bubble stays visibility:hidden while hidden so it can still be measured for the flip). Values are always one of the four cardinal placements — there are no diagonal placements.

4Usage Guidelines

✓ Do

  • Use to label or clarify an icon-only control whose meaning isn't obvious
  • Keep it to a short plain-text phrase — a few words, not sentences of prose
  • Attach it to a genuinely focusable trigger and wire aria-describedby
  • Let the preferred placement auto-flip near edges rather than clipping
  • Rely on all three reveals — hover, focus, and press-and-hold — so touch and keyboard work too

✗ Don't

  • Put links, buttons, or other interactive content inside — use a Popover instead
  • Hide essential information only in a tooltip — it must be discoverable without one
  • Attach it to a non-focusable element (a plain <span>) — keyboard users can't reach it
  • Use it for validation errors or long help text — use inline field help or a Message Box
  • Rely on hover alone — that repeats the legacy failure on touch and keyboard

Under @media (prefers-reduced-motion: reduce) the fade is dropped and the tooltip appears and disappears instantly. The hover intent delay (300ms) and hold delay (500ms) are interaction timings, not animation, and are unaffected.

Acceptance criteria

  • Bubble: inverse surface (colour/surface/inverse), white text (colour/text/inverse), radius/sm, max-width 240px, centred
  • Arrow points to the trigger and follows the placement modifier
  • Reveals on hover after the intent delay, on focus immediately, and on press-and-hold ≥500ms
  • Escape hides it even when shown via hover; hides on pointer-leave and blur
  • Trigger wired with aria-describedby → tooltip id; bubble is role="tooltip"
  • Bubble is pointer-events:none and stays laid out while hidden (for measurement/auto-flip)
  • Preferred placement auto-flips to the opposite side when it would clip its container
  • Plain text only — no interactive content
  • prefers-reduced-motion: fade removed, appears/disappears instantly

9Open Questions

ℹ️
Diagonal placements & collision shift v1.0 supports four cardinal placements with opposite-side auto-flip only. Corner/diagonal placements and along-axis shifting (nudging the bubble to stay in view without flipping) are deferred. If a future layout needs them, they can be added as extra placement modifiers without breaking the existing four.
