Selection Pill

WIP v1.0.1 Updated 2026-06-12

An interactive pill/chip for filtering, tagging, and selection UI. Five states, optional leading icon, and a selectionType prop that switches between no indicator, a dismiss × button, a radio glyph, or a checkbox glyph. The entire pill is the tap target — the radio and checkbox glyphs are decorative.

Related: Pill — non-interactive status/information label.

4Usage Guidelines

✓ Do

  • Use checkbox when multiple filters can be active simultaneously
  • Use radio when only one option can be active at a time
  • Use dismiss for applied filters the user can remove individually
  • Wrap a set of pills in a role="group" with a descriptive label
  • Always provide a visible text label — pills are not icon-only
  • Keep labels short — 1 to 3 words

✗ Don't

  • Mix radio and checkbox in the same pill group
  • Use as navigation or page actions — use a button or nav link instead
  • Use for non-interactive status display — use the non-interactive Pill
  • Place more than ~8 pills in a single group without search or overflow handling
  • Use dismiss for pre-set options the user cannot remove

⚠️
Button-inside-button rule When selectionType='dismiss', the outer pill wrapper must NOT be a <button> element — HTML5 prohibits interactive content inside <button>. Use <li role="option"> or <div role="option"> with keyboard handling instead.

Props interface

Aligned to Figma v1.0
TypeScript 
type SelectionType = 'none' | 'dismiss' | 'radio' | 'checkbox';

interface SelectionPillProps {
  /** Visible text label — always required */
  label:           string;
  /** Show leading decorative icon */
  leadingIcon?:    boolean;       // default: true
  /** Right-side indicator type — also determines ARIA pattern */
  selectionType?:  SelectionType; // default: 'none'
  isSelected?:     boolean;
  isDisabled?:     boolean;
  /** Toggle callback (all selectionTypes) */
  onToggle?:       () => void;
  /** Dismiss callback (selectionType='dismiss' only) */
  onDismiss?:      () => void;
}

Acceptance criteria

  • All 5 states render correctly: default, hover, focus, selected, disabled
  • Height: 44px · radius/full · px: 12px · gap: 4px
  • Default: 1px border action/primary — upgrades to 2px on focus and selected
  • Hover: glow 0 0 1px 4px feedback/success/subtle, border stays 1px
  • Focus: 2px border + same glow, :focus-visible only
  • Selected: surface/green bg, 2px border, no glow
  • Disabled: 2px text-disabled border, text-disabled colour, pointer-events: none
  • selectionType='none': <button aria-pressed>, no indicator
  • selectionType='radio': radio glyph aria-hidden, swaps on selected
  • selectionType='checkbox': checkbox glyph aria-hidden, swaps on selected
  • selectionType='dismiss': outer is NOT <button>; dismiss is <button aria-label="Remove [label]">
  • Dismiss FAB bg tracks pill state per token table in Section 5
  • Leading icon: aria-hidden="true", optional via leadingIcon boolean
  • Group semantics documented and implemented per selectionType

9Open Questions

ℹ️
Dismiss animation When a dismiss pill is removed, an abrupt DOM removal can be jarring. A short fade/scale-out (150ms) at the container level improves perceived quality. This is a layout/page-level concern — the component itself does the onDismiss callback; the parent handles animation.
ℹ️
Overflow handling When many pills are shown in a filter bar, overflow strategy (scroll, wrap, "+N more") is a layout concern outside the component. Document recommended patterns in context-specific guidance.
