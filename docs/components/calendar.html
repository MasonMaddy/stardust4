Component

Calendar

WIP v1.0.0 Updated 2026-07-07

A full month view for a whole service: a hairline-bordered grid of days, each carrying category-coloured event chips, a "+N" badge when a day overloads, weekend shading, and a day-select detail panel. On phones the grid is never shrunk — it collapses to a day-grouped agenda list whose day detail opens the shipped Bottom Sheet. It replaces the legacy pair that shipped on two different foundations — Calendar (a Vue Cal wrapper) and CalendarMonthly (an undocumented, heavily modified Vuetify calendar) — with one component.

ℹ️
Categories are semantic, not new tokens. The three event categories reuse existing families: bookings → feedback success (cyan), staffing → accent (purple), closures → feedback warning (orange). Adding a category means mapping it to an existing family, not minting a colour.

Presentation: phone (agenda + sheet)

A seven-column month grid can't shrink to a phone without becoming unreadable, so on small viewports the calendar switches to .ds-calendar--agenda: the month nav is retained, and every day with events becomes a sticky __agenda-day header grouping its event rows. Tapping a day opens the shipped Bottom Sheet (role="dialog", aria-modal="true") whose list holds that day's events and whose footer carries a Done button; the scrim dismisses and focus returns to the day. The demonstration below is live — the sheet is a fixed overlay contained within the demo phone frame.

4Usage Guidelines

✓ Do

  • Use it when a whole month at a glance is the task — a provider admin scanning a service's bookings, roster changes and closures
  • Keep event labels short — chips truncate, and the full label lives in the day detail
  • Let the "+N" badge and the detail panel carry an overloaded day rather than growing the cell
  • Switch to the agenda presentation on phones — the grid must never be shrunk to fit
  • Map any new event type onto one of the three semantic categories

✗ Don't

  • Reach for the legacy Calendar or CalendarMonthly — they're replaced by this one component
  • Use this to pick a single date in a form — that's the Date Picker
  • Pack sentences into a chip — if it can't be scanned in a couple of words, it belongs in the detail
  • Invent a per-event colour — categories reuse the semantic families, on purpose
  • Shrink the seven columns to phone width — use the agenda presentation instead

Acceptance criteria

  • One month grid backs both the wide panel and the phone agenda; presentation differs, data/logic don't
  • Correct month maths: real weekday offsets, Monday-first by default, today from the system clock
  • WAI-ARIA grid with roving tabindex; arrows/Page/Home/End/Enter as documented; Escape closes the detail
  • Events group by day; a day over maxChips shows a "+N" badge and the count in its aria-label
  • Categories map to the three semantic families — no per-event colours
  • Month nav and agenda-day headers are ≥44px; day cells ≥96px (72px dense)
  • Phone day detail uses the shipped Bottom Sheet; selection survives presentation switches
  • prefers-reduced-motion: colour transitions removed

9Open Questions

ℹ️
Clean & Soft directions, week/day views, and drag The workbench explored three visual directions — Operational, Clean (calm, dot markers) and Soft (card-per-week, gentle depth). v1 ships Operational only (Mason, 2026-07-06); Clean and Soft were exploration directions not carried forward — --clean / --soft are reserved names that would extend this class contract rather than break it. Week and day views, and drag-to-reschedule, are also out of scope for v1.
