Component

Card

WIP v1.1.0 Updated 2026-06-12 DRI: Mason

A full-width row card composing Title Block (avatar + title + subtitle + pills) on the left with a swappable trailing slot on the right. Covers two interaction families — Clickable (whole card is one link/button) and Selectable (whole card is one radio/checkbox/switch) — plus a plain non-interactive Container.

ℹ️
Selected ≠ focus (review D1) Selected is a persistent value indicator: 2px action/primary border + surface/cyan fill + action/primary title, no halo — matching the multi-select dropdown selected-row pattern. The success-subtle halo stays reserved for :focus-visible, so a keyboard user can tell "this card has focus" apart from "this card is selected" even when both apply at once.
ℹ️
No layout shift on select — inset-ring trick The default card keeps its 1px border/default. Selecting sets border-color: action/primary plus box-shadow: inset 0 0 0 1px action/primary — the inset ring stacks on the existing 1px border to read as a visual 2px without changing the box size, so surrounding cards never jump. (Flat inset shadow — not elevation.)

4Usage Guidelines

Use this component when

  • A row navigates somewhere or triggers one action — use Clickable (whole card is the link/button)
  • A row is a choice — pick-one (radio), pick-many (checkbox), or an immediate on/off setting (switch) — use Selectable
  • A row presents an entity with its own actions (text action or icon-button row) — use Container

Don't use this component when

  • Dense, scannable data — long tables or simple lists read better as plain list rows; the card's 20px padding and border are for standalone, tappable rows
  • The row needs both a whole-card action and inner buttons — split it: Container with explicit controls, or Clickable with the secondary actions moved elsewhere
  • Free-form content blocks — this card is the Title Block + trailing slot row pattern, not a generic panel

Do / Don't

✓ Do
Whole-card control or container with inner controls — never both. The selectable card is one control: the glyph is visual-only, one tab stop, full-card hit area.
✗ Don't
Never nest interactive elements inside a clickable or selectable card — a button inside a link is invalid HTML and breaks screen-reader output (review D2).

Content

  • Title is the card's accessible name — keep it unique within a group
  • Subtitle carries supporting context (relationship, room, status), not a second action
  • Pills are non-interactive metadata (ds-pill --sm --green --minimal) — never use them as buttons

6Accessibility

Semantics & keyboard (review D2)

VariantElement & ARIAKeyboard
clickableSingle <a> (navigation) or <button> (action). Whole card is the hit area, one tab stop. Chevron aria-hiddenEnter (+ Space for button)
radioCard div role="radio" aria-checked tabindex per roving-tabindex group inside role="radiogroup"; glyph aria-hiddenArrows move+select within group, Space selects
checkboxCard div role="checkbox" aria-checked tabindex="0"; glyph aria-hiddenSpace toggles
switchCard div role="switch" aria-checked tabindex="0"; toggle visuals aria-hiddenSpace (and Enter) toggles, takes effect immediately
containerPlain <div> — no role, not focusable. Inner buttons/links are the interactive elementsInner controls' native behaviour
🏷️
Accessible name
The card control's name comes from the title: aria-labelledby → title element id; subtitle via aria-describedby if needed.
🙈
Glyphs are visual-only
The radio/checkbox/toggle glyphs and the chevron carry aria-hidden="true" and use visual classes only — state is announced from the card's aria-checked, not the glyph.
🚫
No nested interactives
Never nest interactive elements inside the clickable/selectable card (invalid HTML, broken screen-reader output). Use the container family when inner controls are needed.
📐
Touch target
The whole card is the target (≥104px tall at default content) — far exceeds the 44px minimum. The compact variant holds a min-height: 44px floor and grows with content.
⚠️
Compact glyph collapse — whole-card controls only
.ds-card--compact collapses the radio/checkbox glyphs' 44×44px hit-area wrappers to their 24px visual size. That is safe only because the whole card is the control and itself holds the 44px min-height touch floor — never use the --compact glyph collapse with standalone checkboxes/radios, where the wrapper is the touch target.

Props interface

Aligned to spec v0.1
TypeScript 
type CardVariant = 'clickable' | 'radio' | 'checkbox' | 'switch' | 'container';

interface CardProps {
  variant?:    CardVariant;  // default: 'container'
  /** Selectable family only (radio | checkbox | switch) — review D3.
      Not valid for clickable/container. */
  selected?:   boolean;
  /** Clickable + selectable families */
  isDisabled?: boolean;
  /** Dense lists and sheet rows — applies .ds-card--compact:
      6px/12px padding, 14px/20px type, collapsed glyph hit-wrappers
      (safe only because the whole card is the control), 44px
      min-height touch floor — height grows with content. */
  compact?:    boolean;
  /** clickable only: renders <a> (navigation) when set, <button> (action) otherwise */
  href?:       string;

  // Title Block pass-through — rendered by ds-title-block per its own spec
  title:       string;       // becomes the card's accessible name (aria-labelledby)
  subtitle?:   string;
  pills?:      string[];     // ds-pill --sm --green --minimal
  avatar?:     AvatarProps;  // ds-avatar --square --image, 64px

  /** container only: free trailing content (minimal text action /
      ds-btn --ghost --icon-only row). Clickable renders the chevron and
      selectable renders the glyph internally. */
  trailing?:   Slot;

  /** clickable */
  onPress?:    () => void;
  /** selectable */
  onChange?:   (selected: boolean) => void;
}

Acceptance criteria

  • All 5 variants render on web, iOS, and Android composing existing DS pieces only (avatar, title block, pill, button, radio/checkbox/toggle visuals)
  • D1: selected = 2px action/primary border + surface/cyan fill + action/primary title, no halo; halo reserved for :focus-visible
  • D1: selecting causes zero layout shift — 1px border + inset 1px ring reads as 2px
  • D1: hover/pressed while selected → surface-cyan-hover
  • D2: clickable/selectable card is a single control — one tab stop, full-card hit area, glyph/chevron aria-hidden, no nested interactives
  • D2: aria-labelledby wires the card control to its title element
  • Radio cards: roving tabindex inside role="radiogroup"; arrows move+select, Space selects
  • Checkbox cards: Space toggles, independent state
  • Switch cards: role="switch", Space (and Enter) toggles with immediate effect
  • Container cards: no role, not focusable; inner controls keep native behaviour
  • Hover surface/grey, pressed grey/300, focus border/focus + success-subtle halo; flat throughout — no elevation
  • Disabled: content → text-disabled, cursor: not-allowed, not focusable
  • All token values match Section 5 — no new tokens

9Open Questions

Intentional design decisions worth confirming before engineering finalises tokens and behaviour. Not blockers for the current spec — flag in design review.

⚠️
1 · Pressed bg uses the grey/300 primitive The pressed tint references colour/grey/300 directly rather than a semantic token. Consider a semantic alias (e.g. surface/grey-pressed) alongside surface-cyan-hover before tokens are finalised.
⚠️
1b · Pills disappear on selected cards On a selected card the ds-pill --green --minimal pills (surface/cyan bg) sit on the surface/cyan card fill — the pill shape disappears, only its text reads. Options: accept (dropdown rows never carried pills), switch selected-card pills to the solid green variant, or give pills a border on cyan surfaces. Visible in the Selected demos above — needs a design call.
ℹ️
2 · Active/current-page treatment for clickable cards selected was dropped for the clickable family (review D3). Does navigation need an active/current-page treatment to replace it?
ℹ️
3 · Radio-card group focus model Roving tabindex vs all-tabbable — the spec says roving (matches ds-radio), as implemented in the Section 2 demo. Confirm with engineering.
ℹ️
4 · Figma follow-ups (design)
  • TitleBlock instance inside ClickableCard has drifted typography (subtitle 12px vs spec 14px Subtitle/Regular) — align to the Title Block component (D4/D5)
  • Remove the now-deprecated selected variants for type = icon | text | buttons once instances are migrated (D3)
  • Switch set has a stray case-duplicate value State=Off alongside off — merge
  • Chevron drawn at 25px — swap to standard 16px icon
