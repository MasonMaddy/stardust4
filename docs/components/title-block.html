Component

Title Block

WIP v1.2.0 Updated 2026-06-12

A horizontal layout block combining a square Avatar with a content column (title, subtitle, 0–N non-interactive pills). Used as a header pattern for profile cards, list items, and detail views. Fully non-interactive — the parent handles tap/click if needed.

Assembles: Avatar · Pill

4Usage Guidelines

Avatar size — match to content height

The avatar size is chosen by the designer/engineer to visually align with the content column height. There is no automatic sizing — select the --avatar-size that best matches.

Content shownApprox heightRecommended avatar
Title + subtitle + pills~79px64px
Title + subtitle~51px56px
Subtitle + pills~51px56px
Title + pills~52px44px
Title only~24px32px
Subtitle only~23px32px

✓ Do

  • Choose the avatar size that visually matches the content column height
  • Use 64px avatar when showing the full combination (title + subtitle + pills)
  • Use 56px avatar when showing title + subtitle or subtitle + pills
  • Use 32px avatar when showing a single row of content
  • Use consistent avatar sizes across all items in a list
  • Provide descriptive alt text on avatar images
  • Wrap in a parent element to handle click/navigation — Title Block itself is non-interactive
  • Apply max-width on .ds-title-block when truncation is needed

✗ Don't

  • Use a large avatar (64px) with only a title — creates awkward empty space
  • Use a small avatar (32px) with full content (title + subtitle + pills) — avatar appears misaligned
  • Mix avatar sizes within the same list
  • Use more than 3–4 pills — they wrap and make the component unpredictable in height
  • Add interaction (hover/focus/press) to the Title Block itself — use the parent container
  • Omit alt text on avatar images

In a card list context

The Title Block sits inside a padded row. The row itself handles interactivity (hover state, tap target, navigation). Title Block provides only layout and content.

6Accessibility

RequirementImplementation
Non-interactive elementRender as <div> — no role, no tab stop. The parent row/card handles interactivity.
Heading semanticsThe title maps to a context-appropriate heading level (<h2>/<h3>… depending on where the block sits in the page outline) — never a fixed level baked into the component. In list rows a plain <p> is fine; a heading per row adds outline noise.
Avatar imageMust have descriptive alt text (e.g. the child's name). Initials/icon avatars: aria-label on the avatar element. When the avatar is decorative and sits next to the visible name (the common case), use empty alt="" instead — a named alt would double-announce the person to screen readers.
PillsRendered as non-interactive <span> elements and inherit Pill's accessibility guidance — see Pill · Accessibility. aria-hidden is not needed on the pill itself — pill text is meaningful context — but decorative leading icons and trailing flags inside a pill take aria-hidden="true".
TruncationThe ellipsis is visual-only: the full text stays in the DOM, so screen readers always announce the complete value. Optionally add title="[full text]" to surface the full value on pointer hover. Never trim the string itself before rendering.
Parent interactionWhen the Title Block is inside a tappable row, the parent <button> or <a> carries the accessible name (e.g. aria-label="View Jamie Anderson's profile").

7Motion

ℹ️
No motion on the Title Block itself. It is a non-interactive display component. Any transition (e.g. row tap feedback, skeleton loading) is handled by the parent container.

Acceptance criteria

  • Container: inline-flex, align-items: center, gap 12px
  • Avatar: ds-avatar--square, fixed size via --avatar-size, flex-shrink: 0
  • Avatar size chosen to match content column height per sizing guide in Section 4
  • Content column: flex-direction: column, gap 4px, min-width: 0
  • Title: 20px / 24px lh / Medium 500 (default) or SemiBold 600 · colour/text/text-primary
  • Subtitle: 14px / 23px lh / Regular 400 (Subtitle/Regular) · colour/text/text-primary
  • margin: 0 on title and subtitle — overrides main.css p { margin-bottom: 0.9rem } (14.4px)
  • Title and subtitle: white-space: nowrap; overflow: hidden; text-overflow: ellipsis — truncates when max-width is applied to .ds-title-block
  • Pills: ds-pill--sm ds-pill--green ds-pill--minimal, gap 4px, flex-wrap
  • Pills row rendered only when pills.length > 0
  • showAvatar, showTitle, showSubtitle independently toggleable
  • Non-interactive: no hover, focus, or pressed styles on the component itself

9Open Questions

ℹ️
Pill colours The current spec uses only the green minimal pill inside Title Block. In practice, status pills (e.g. Urgent, Overdue) may use orange or grey variants. Confirm whether Title Block pills are always green minimal or whether other Pill colours should be supported.
ℹ️
Pills overflow strategy When many pills are shown, flex-wrap increases the component height (and thus the avatar height recommendation). For dense lists, consider a max pill count (2–3) and truncate with a "+N more" badge. Not currently specced.
