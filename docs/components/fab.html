Component

FAB

WIP v1.1.0 Updated 2026-06-12

A circular icon action button — four colour types, four sizes, six states. The xl size supports an optional text label, covering the Extended FAB pattern from MD3. All sizes are interactive; accessibilityLabel is required when no visible text label is present.

4Usage Guidelines

✓ Do

  • Use xl for the primary action on a page or view
  • Add a label (showText=true) on xl when the action needs context
  • Use lg (44px) for contextual actions that need a standard touch target
  • Use md/sm for secondary inline actions in dense layouts
  • Use filled when placing the FAB on a coloured or image background
  • Always provide accessibilityLabel for icon-only variants

✗ Don't

  • Use more than one xl FAB per view — it signals the primary action
  • Use sm (24px) on touch/mobile — below the 44px WCAG tap target
  • Use the FAB for navigation — use a nav component instead
  • Place a FAB inside scrollable content unless it's fixed/sticky positioned
  • Use without an icon — the icon is required

6Accessibility

RequirementImplementation
Native controlAlways render as <button type="button"> — never a <div> or <a>. Native semantics provide role, focusability, and keyboard activation for free
KeyboardTab to focus · Enter / Space to activate — native <button> behaviour, no extra key handling required
Focus indicator2px solid type-specific border on :focus-visible only — never on hover or click. outline: none is set but replaced by the border. Note: Button uses an outer glow; the FAB uses a border instead
Touch targetlg (44px) and xl (56px) meet the WCAG 2.1 SC 2.5.5 44×44px target intrinsically. sm (24px) and md (32px) do not — restrict to desktop-only surfaces or extend the hit area (e.g. padded wrapper / pseudo-element) to 44×44px
Fixed xl placementWhen the xl FAB is fixed/floating it must not obscure content or focus targets for keyboard and screen-magnifier users; respect env(safe-area-inset-*) on notched mobile displays
Icon-only (sm/md/lg, xl without label)Must have aria-label or aria-labelledby on the <button> element
xl with showText=trueVisible text label satisfies the accessible name — no extra aria-label needed
Icon slotaria-hidden="true" — decorative, never the accessible name
Disabled stateUse disabled attribute. pointer-events: none is applied automatically
Colour aloneDon't rely solely on type colour to convey meaning; pair with a label or contextual placement

Props interface

Aligned to Figma v1.0
TypeScript 
type FabType  = 'default' | 'secondary' | 'tertiary' | 'inverse';
type FabSize  = 'sm' | 'md' | 'lg' | 'xl';

interface FabProps {
  type?:               FabType;   // default: 'default'
  size?:               FabSize;   // default: 'xl'
  isDisabled?:         boolean;
  /** xl only — show text label alongside icon (Extended FAB) */
  showText?:           boolean;
  /** label text when showText=true */
  text?:               string;
  /** Required when showText=false — provides accessible name */
  accessibilityLabel?: string;
  /** Icon element. Any icon from iconOutline or iconColoured. */
  icon?:               ReactNode;
  onPress?:            () => void;
}

Acceptance criteria

  • All 4 types × 4 sizes × 6 states render correctly across web, iOS, Android
  • Sizes: sm=24px, md=32px, lg=44px, xl=56px height
  • radius/full (9999px) — fully circular on all sizes and states
  • Icon sizes: sm=12px, md=20px, lg=22px, xl=28px — all aria-hidden="true"
  • colour/focus/primary (#7BCAC5) used as hover background for default type
  • Focus: 2px solid border, type-specific colour, :focus-visible only, no box-shadow
  • Pressed: full saturated token per type, white icon/text colour
  • Disabled: colour/action/disabled bg, colour/text/text-disabled icon/text, all 4 types identical
  • Filled: fully saturated bg per type, white icon/text
  • xl showText=true: icon + 8px gap + 20px/24px Inter Medium label, width hugs content
  • Icon-only variants: accessibilityLabel required (enforced or documented)
  • sm: documented as desktop-only (24px below WCAG 44px touch target)
  • No elevation or box-shadow on any variant

9Open Questions

ℹ️
Tertiary hover invisible on default bg Tertiary default and hover backgrounds are both colour/surface/orange / colour/feedback/warning/subtle (#FEF2E0). The hover state is perceptually invisible without cursor feedback. On touch devices there is no hover — consider if tertiary needs a more visible hover differentiator in a future update.
ℹ️
xl as fixed/floating element The xl FAB is designed for prominent page-level actions and is commonly positioned as position: fixed at the bottom-right of a view. Positioning, z-index, and safe-area inset handling (for mobile notched displays) are layout concerns outside the component — document usage patterns in context-specific guidance.
