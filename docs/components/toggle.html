Toggle

WIP v1.0.1 Updated 2026-06-10

A binary on/off switch. A 40×24px visual control — a 36×12px rail with a 20px circular knob that slides 20px between off (left) and on (right). The label is always external; associate it in the parent layout via for/id.

4Usage Guidelines

✓ Do

  • Use for a single setting that takes effect immediately on change
  • Always pair with an external label that describes the controlled setting
  • Position the label to the right or above — never embed it in the component
  • Use in settings screens and preference panels
  • Show the current state clearly — the rail and knob colour communicate on/off

✗ Don't

  • Use when a confirmation step is required before the action takes effect — use a Button instead
  • Use for multiple-option selection — use Checkbox or Radio Button
  • Omit a label — a toggle without context is inaccessible
  • Use for actions that are hard to undo (e.g. deleting data) — use a Button with a confirmation dialog
  • Place toggles inline within running text

Acceptance criteria

  • Visual: 40×24px container — rail 36×12px radius/m, thumb 20×20px radius/full
  • Touch target: ≥44×44px click/tap area; visual stays 40×24px
  • Off: grey rail (border/strong), white thumb (action/foreground), thumb center at 7.5px
  • On: cyan-500 rail (feedback/success/default), cyan-700 thumb (action/primary), thumb center at 27.5px
  • Rail depth: on-state rail #01A39D intentionally lighter than knob #00776B
  • Thumb travels 20px with 200ms cubic-bezier(0.2,0,0,1)
  • Rail background-color transitions simultaneously with thumb
  • Ripple: ::before 29px — feedback/success/subtle 70% hover, text-disabled 70% pressed, 40% default
  • Focus off: white thumb + 2px action/primary border, :focus-visible only
  • Focus on: action/primary thumb + 2px action/hover border
  • Disabled: action/disabled rail, surface/grey thumb at 70% — same for on and off
  • role="switch" with aria-checked="true/false"
  • No label in component — external label associated via for/id
  • prefers-reduced-motion: all transitions disabled

9Open Questions

ℹ️
Icon on thumb MD3 Switch supports an optional icon inside the thumb to reinforce the on/off state (e.g. ✓ when on). Stardust Toggle v1.0 does not include this. If a visual reinforcement beyond colour is needed for accessibility or brand reasons, this can be added as a future enhancement without breaking the existing variant matrix.
