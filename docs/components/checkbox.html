Component

Checkbox

Published v1.0.0 Updated 2026-06-03 DRI: Mason

Control-only checkbox for independent multi-select and boolean toggles. Three types — checked, unchecked, and indeterminate — across six interaction states inside a fixed 44×44px hit area. Labels are composed in parent layouts, not in this component.

3Usage

Interactive demo

Motion follows Material Design 3 checkbox motion guidelines. State layer scales from scale(0) with Standard easing (200ms in, 150ms out via Standard Accelerate). Check draws via stroke-dashoffset with Emphasized Decelerate easing (150ms, 50ms delay). Press applies a box squeeze (scale(0.85), 75ms). All transitions respect prefers-reduced-motion.

Label composition

This component has no hasLabel variant. Wrap control + text in a parent <label> with gap: spacing/stack-gap/default (8px). Helper and error text sit beside the label, linked via aria-describedby.

Acceptance criteria

  • All 18 Figma variants (3 types × 6 states) render correctly on web, iOS, and Android
  • 44×44px / 44pt / 48dp hit area on every variant; 18px / 18pt / 18dp visual box centred
  • Hover, focus, pressed, disabled, and error states match token matrix in Section 4
  • Focus ring is circular halo (radius/full) — not Button's inner-border glow
  • Focus ring appears on :focus-visible / keyboard navigation only, never on mouse click
  • Native <input type="checkbox"> always present (visually hidden, not removed)
  • indeterminate sets aria-checked="mixed"; first user click clears it and sets checked
  • Error state sets aria-invalid="true" and links to error message via aria-describedby
  • prefers-reduced-motion disables all transition animations
  • Component ships without a label — external <label> / aria-labelledby required

7Open Questions

Decisions worth confirming before engineering finalises implementation.

✓
1 · Naming resolved — canonical name is indeterminate The engineering and documentation name is indeterminate throughout (matching HTML, ARIA, and all platform APIs). The Figma layer will be updated from intermediate to indeterminate to align. Closed.
✓
2 · Unchecked focus halo colour resolved — darkened to grey-800 Resolved 2026-06-11. The unchecked focus halo deliberately differs from the cyan-tinted checked/indeterminate halo, and colour/focus/secondary was darkened from grey-600 (#BDBDBD) to grey-800 (#838383) to meet WCAG 2.4.11 Focus Appearance (~3.3:1 against white). The Figma variable was updated the same day to match. Closed.
ℹ️
3 · Label composition ownership Labels are deliberately out of scope for this component. Confirm whether a CheckboxField wrapper (checkbox + label + optional helper/error text) is a separate component on the roadmap, or whether teams compose it ad-hoc in their own layouts.
