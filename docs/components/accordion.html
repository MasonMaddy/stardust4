Component

Accordion

WIP v1.0.0 Updated 2026-07-07

A vertical stack of expand/collapse sections for progressively disclosing content. Each item is a real <button> header inside a heading, wired to the panel below it. Height animates via CSS grid-template-rows (0fr→1fr) — content-height agnostic, with no JavaScript measuring. It replaces the legacy Vuetify ExpansionPanels, which was styling-only with no behaviour of its own; the Accordion owns the behaviour — animated height, single/multiple expand, and a full keyboard contract.

4Usage Guidelines

✓ Do

  • Use to progressively disclose long or secondary content that isn't all needed at once
  • Write short, scannable, distinct section titles — the header is the only thing shown collapsed
  • Use single mode when only one section is relevant at a time; multiple when users compare across sections
  • Compose shipped components (forms, checkboxes) inside a panel body — they work unchanged
  • Use --separated when the sections are independent objects rather than parts of one whole

✗ Don't

  • Hide content a user needs to complete their primary task behind a collapsed header
  • Nest accordions deeply — reconsider the information architecture first
  • Use it for step-by-step flows — use a stepper or a wizard instead
  • Use it as primary navigation between pages — that's the nav/tabs' job
  • Reintroduce the legacy light-purple separated literal — --separated is on Stardust tokens

6Accessibility

Touch targetHeader min-height:48px — exceeds the 44×44 minimum in every size
StructureEach header is a real <button> inside a <h4 class="ds-accordion__heading"> (WAI-ARIA APG accordion pattern)

PanelPanel is associated to its header via the header's aria-controls (pointing at the panel id). The optional role="region" / aria-labelledby pairing is intentionally omitted — APG discourages it when several panels are present, to avoid an excess of landmarks
Hidden contentCollapsed panels are made inert so their content is skipped by focus and the accessibility tree during and after the animation

Acceptance criteria

  • Header is a <button> in an <h4>; ≥44px touch target (48px min-height)
  • Height animates via grid-template-rows 0fr↔1fr — no JS height measuring; content of any height opens correctly
  • Chevron rotates 180° when aria-expanded="true"
  • Single mode keeps one panel open; multiple mode allows any number
  • Enter/Space toggle; Arrow ↑/↓ move between headers (wrapping); Home/End jump to first/last
  • aria-expanded + aria-controls on the header wire it to the panel id (region role omitted per APG to avoid excess landmarks)
  • Collapsed panels are inert so their content is unreachable
  • Disabled items use aria-disabled (stay focusable), not the disabled attribute
  • --separated uses colour/border/default + radius/m — never the legacy light-purple literal
  • prefers-reduced-motion: all transitions removed; state changes are instant
