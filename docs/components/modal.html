Component

Modal

WIP v1.0.0 Updated 2026-07-08

A modal surface that interrupts the flow for a focused decision or short task. A flat panel centred over a scrim, with a title, an internally-scrolling body, and a trailing action bar. Comes in four sizes and two modes — dismissible and persistent (a blocked dismiss nudges the panel rather than silently doing nothing). One component replaces the legacy CommonModal and CommonPersistentModal. On phones the same content presents as the shipped Bottom Sheet instead.

ℹ️
Persistent is a mode, not a variant. One modal replaces both legacy modals. In persistent mode a blocked dismiss (backdrop click or Escape) plays a gentle attention nudge (.is-nudged) instead of silently ignoring the click — a visible “finish this first” cue the legacy persistent modal never gave.

4Usage Guidelines

✓ Do

  • Use for a focused decision or short task that must interrupt the current flow
  • Keep the title a clear question or statement and the body concise
  • Put the primary action last (trailing) with a clear cancel/dismiss alongside
  • Reserve persistent mode for genuinely blocking steps the user must complete
  • On phones, present the same content as a Bottom Sheet — thumb-reachable

✗ Don't

  • Stack modals on top of each other — resolve one before opening another
  • Use persistent mode as a dark pattern to trap users in a flow they want to leave
  • Put long-form content or full workflows in a small modal — use a full page instead
  • Use a modal for transient, non-blocking feedback — use a Toast
  • Omit a way out — dismissible modals always close on Escape and backdrop click

Acceptance criteria

  • Panel: colour/surface/default, radius/lg, centred over the colour/scrim backdrop
  • Sizes cap max-width at 400 / 560 / 800px; --fullscreen fills the viewport with no radius or padding
  • Body scrolls internally; title and actions stay pinned
  • Panel is role="dialog" aria-modal="true" + aria-labelledby; takes initial focus
  • Tab is trapped inside the panel; focus returns to the opener on close
  • Dismissible: Escape / backdrop / action closes. Persistent: blocked dismiss nudges (.is-nudged), modal stays open
  • Below 640px the same content presents as the Bottom Sheet; kept-modal actions stack full-width
  • prefers-reduced-motion: enter animation and nudge removed; behaviour unchanged
  • Replaces legacy CommonModal and CommonPersistentModal

9Mobile Presentation

Below the mobile breakpoint a centred modal is a poor fit — it's hard to reach one-handed and fights the on-screen keyboard. The same title, body and actions present instead as the shipped Bottom Sheet (.ds-sheet, reused from bottom-sheet.css — the modal does not re-declare those rules). This is a presentation-swap, not different content: same decision, thumb-reachable surface.
