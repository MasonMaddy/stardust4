Icons

Published v1.0.0 2 sets · 278 icons Synced from Figma 2026-06-02

Stardust ships two icon sets: iconOutline — monochrome stroke icons for standard UI controls and actions — and iconColoured — illustrated full-colour icons for domain-specific features such as care categories, health, nutrition, and activities.

6Usage

Colour — iconOutline

Outline icons are monochrome. Apply a single fill token to the SVG. Never hardcode hex values — always reference a --sd-colour- token.

ContextTokenCSS varValue
Default (primary action)colour/action/primary--sd-colour-action-primary#00776B
Hovercolour/action/hover--sd-colour-action-hover#008480
Pressedcolour/action/pressed--sd-colour-action-pressed#004B40
On dark fill (button icon)colour/text/text-inverse--sd-colour-text-inverse#FFFFFF
Disabledcolour/text/text-disabled--sd-colour-text-disabled#BDBDBD
Secondary / supportingcolour/text/text-secondary--sd-colour-text-secondary#838383

Colour — iconColoured

⚠️
Do not recolour iconColoured icons. The colours are embedded in the icon and carry semantic meaning (e.g. red heart = favourite, teal check = complete). Applying a CSS fill or opacity change to a coloured icon will break its intent. Use them at their native colours only.

Do / Don't

✓ Do
Use iconOutline for interactive controls and always pair with a visible label or aria-label.

Use iconColoured for feature-identity contexts where the illustrated style is appropriate (activity cards, category tiles, health summaries).
✗ Don't
Don't use iconColoured inside buttons, form fields, or interactive controls.

Don't scale icons to non-standard sizes (e.g. 18px, 22px) — use the 16 / 20 / 24 / 32 scale only.

Don't use icon-only controls without an aria-label or tooltip.

7Accessibility

🔊
Decorative icons — hide from screen readers
When an icon accompanies a visible text label, mark it aria-hidden="true". The label already conveys the meaning — announcing the icon adds noise.
Web <svg aria-hidden="true">  iOS .accessibilityHidden(true)  Android ImportantForAccessibility.No
🏷️
Icon-only controls — require a label
When an icon is the only visible affordance (no adjacent text), provide an accessible name.
Web <button aria-label="Search"> or <title> inside SVG
iOS .accessibilityLabel("Search")
Android contentDescription = "Search"
🎨
Contrast — iconOutline
action/primary (#00776B) on white = 4.65:1 — passes WCAG 2.1 AA (3:1 for graphical elements). text/secondary (#838383) on white = 3.95:1 — passes AA for non-text graphics. Disabled icons are intentionally low contrast — WCAG 1.4.3 exempts disabled controls.
⛔
Never rely on colour alone
For iconColoured, do not use colour as the only differentiator of meaning. Pair coloured icons with a text label or category name so that users with colour vision deficiencies can still distinguish them.

Props interface

TypeScript 
type IconSet  = 'iconOutline' | 'iconColoured';
type IconSize = 16 | 20 | 24 | 32;

interface IconProps {
  /** Icon set — determines which SVG sprite to reference */
  set: IconSet;
  /** Icon name within the set, e.g. 'action/search' */
  name: string;
  /** Render size in px — must be one of the standard steps */
  size?: IconSize;
  /**
   * Accessible label.
   * Required when the icon has no adjacent visible text label.
   * Pass undefined and aria-hidden="true" for decorative icons.
   */
  label?: string;
  /** CSS colour token for iconOutline only — ignored for iconColoured */
  colour?: string;
}

Acceptance criteria

  • Both icon sets render from a single sprite/asset per platform
  • iconOutline colour is driven by a token — no hardcoded hex in component code
  • iconColoured renders at original colours — no tint applied
  • Icons at 16 / 20 / 24 / 32px produce crisp output on 1×, 2×, 3× displays
  • Decorative icons carry aria-hidden="true" / equivalent
  • Icon-only controls carry an accessible label
  • Name resolution matches the iconOutline/category/name convention

9Open Questions

⚠️
1 · Exact icon layer names need verification Names in the category index above are approximated from the Figma screenshot. Before engineering implements the sprite, open each Figma frame and confirm the exact layer names — they will be the stable reference in code.
⚠️
2 · SVG export pipeline not yet established Individual SVG files have not been exported from Figma and added to the repo. Decide on the delivery format (sprite vs individual files vs icon font vs component library) before engineering begins.
ℹ️
3 · iconColoured interactive states not defined The spec does not define hover/pressed/disabled states for coloured icons. Confirm whether animated or state-variant coloured icons are needed, or whether coloured icons are always static/non-interactive.
ℹ️
4 · iOS asset catalog vs SF Symbols Where an iconOutline icon has a direct SF Symbols equivalent, using the system symbol provides better Dynamic Type scaling and accessibility support. Decide on a policy: custom SVGs only, or SF Symbols where equivalent, custom elsewhere.
