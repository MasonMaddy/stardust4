Icons

WIP v1.1.0 282 icons · 16 pictograms

Stardust ships two icon sets plus a pictogram set: iconOutline (145 monochrome stroke icons for UI actions and navigation), iconColoured (137 illustrated icons for health, care, and feature contexts), and Pictograms (16 larger illustrated images for empty states, loading, banners, and other helper spaces). Click any icon to copy its name; hover to download the SVG file.

Sizing

iconOutline icons are designed on a 16×16 grid. Scale up using multiples; avoid non-integer sizes on screen.

16px spacing/4 Inline with text, dense UI
20px ★ spacing/5 Default — most buttons, lists, nav
24px spacing/6 Feature callouts, empty states
32px spacing/8 Large feature icons, onboarding

iconColoured: Do not override the colour fills of iconColoured icons. The multi-colour fills are part of the visual language for health and care contexts. Scale size only.

✓ Do

  • Use iconOutline for all interactive UI elements (buttons, nav, actions)
  • Set aria-hidden="true" on decorative icons
  • Use iconColoured for health events, nutrition, emotional states
  • Match icon size to adjacent text size (1.25× ratio)

✗ Don't

  • Override the colour fills of iconColoured icons
  • Use iconColoured icons in button or navigation contexts
  • Show icon-only interactive elements without a visible label or tooltip
  • Use non-integer sizes (e.g. 18px, 22px)

Accessibility

RequirementDetail
Decorative icons Always add aria-hidden="true" and no alt text. The label comes from adjacent text or the parent button.
Icon-only interactive Add aria-label or title to the parent element. Ensure a visible tooltip is shown on hover/focus.
Contrast (iconOutline) Primary fill #252525 on white = 15.2:1 ✓ AAA. Action colour #00776B on white = 4.65:1 ✓ AA. Always check contrast for non-standard colours.
Touch target Icon-only buttons must meet 44×44px minimum. Pad around the icon SVG in the button container.
SVG title For standalone meaningful SVGs, add a <title> element inside the SVG and reference it with aria-labelledby.

</button>

Props interface

Aligned to Figma v1.0
TypeScript 
type IconSet  = 'iconOutline' | 'iconColoured';
type IconSize = 16 | 20 | 24 | 32;

interface IconProps {
  /** Which icon set to render from */
  set: IconSet;
  /** Kebab-case file name without extension, e.g. 'search', 'notification-bell' */
  name: string;
  /** Rendered size in px — defaults to 20 */
  size?: IconSize;
  /** Accessible label — required when icon is the sole visible content of an element */
  label?: string;
  /** CSS colour override — iconOutline only; never pass for iconColoured */
  colour?: string;
}

SwiftUI 
import SwiftUI

enum StardustIconSet { case outline, colour }

struct StardustIcon: View {
    var set:    StardustIconSet
    var name:   String
    var size:   CGFloat = 20
    var label:  String? = nil
    var colour: Color?  = nil

    var body: some View {
        // Asset name convention: "outline/search" or "colour/happy"
        let assetName = "\(set == .outline ? "outline" : "colour")/\(name)"

        Group {
            if set == .outline {
                // iconOutline — template rendering so tint applies
                Image(assetName, bundle: .module)
                    .resizable()
                    .renderingMode(.template)
                    .foregroundColor(colour ?? Color(UIColor.label))
            } else {
                // iconColoured — original rendering; fills must not be overridden
                Image(assetName, bundle: .module)
                    .resizable()
                    .renderingMode(.original)
            }
        }
        .scaledToFit()
        .frame(width: size, height: size)
        .accessibilityLabel(label ?? "")
        .accessibilityHidden(label == nil)
    }
}

import androidx.compose.foundation.layout.size
import androidx.compose.material3.Icon
import androidx.compose.material3.LocalContentColor
import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import androidx.compose.ui.graphics.Color
import androidx.compose.ui.graphics.ColorFilter
import androidx.compose.ui.res.painterResource
import androidx.compose.ui.unit.Dp
import androidx.compose.ui.unit.dp

enum class StardustIconSet { Outline, Colour }

@Composable
fun StardustIcon(
    set:      StardustIconSet,
    name:     String,
    modifier: Modifier = Modifier,
    size:     Dp       = 20.dp,
    label:    String?  = null,   // null = decorative (contentDescription omitted)
    colour:   Color?   = null,   // iconOutline only; ignored for iconColoured
) {
    // Drawable naming: ic_outline_search or ic_colour_happy
    val prefix  = if (set == StardustIconSet.Outline) "ic_outline_" else "ic_colour_"
    val resName = "$prefix${name.lowercase().replace("-", "_")}"
    val resId   = /* LocalContext.current.resources.getIdentifier(
                     resName, "drawable", LocalContext.current.packageName) */
        android.R.drawable.ic_dialog_info // replace with real resolution

    Icon(
        painter            = painterResource(resId),
        contentDescription = label,
        modifier           = modifier.size(size),
        tint               = if (set == StardustIconSet.Outline)
                                 colour ?: LocalContentColor.current
                             else
                                 Color.Unspecified, // preserves embedded fills
    )
}

Acceptance criteria

  • All 145 iconOutline SVGs render at 16, 20, 24, and 32px without distortion
  • All 137 iconColoured SVGs render with embedded fills intact — no tint applied
  • iconOutline colour is overrideable via CSS color / platform tint; defaults to current text colour
  • iconColoured colour is never overridden by the component
  • Decorative icons are hidden from assistive technology (aria-hidden / contentDescription = null)
  • Meaningful icons expose a visible or programmatic label
  • size defaults to 20px; valid values are 16, 20, 24, 32 — no non-integer sizes
  • Icon-only interactive elements meet 44×44px / 44pt / 48dp minimum touch target

Open questions

Decisions worth confirming before engineering integration.

⚠️
1 · Delivery format SVG sprite, individual SVG files, or compiled component library? This determines the engineering integration pattern and affects how icons are referenced in code. Individual SVG files are already exported — confirm whether a sprite or component wrapper is needed before implementation begins.
⚠️
2 · iconOutline fill convention Exported SVGs currently use fill="#252525" (Figma export default). For CSS theming, fill="currentColor" is recommended so icons inherit text colour automatically. Confirm and update SVGs before engineering integration.
⚠️
3 · Figma naming typos Dowload-dash-arrow (missing 'n' — exported as dowload-dash-arrow.svg) and knifr-fork (typo — exported as knifr-fork.svg). Confirm correct names and re-export before finalising engineering references.
⚠️
4 · Duplicate iconColoured name "Expired" Two Figma nodes share the name "Expired" (3862:164332 and 3862:167089). The second has been exported as expired-2.svg. Confirm the intended name for each before engineering ships.
ℹ️
5 · Icon categorisation Icons are currently flat (no category nesting in Figma). A categorisation system (e.g. iconOutline/action/search) is needed for the engineering API if icons are to be discoverable by category. Confirm the taxonomy before building the component API.
