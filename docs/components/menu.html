Component

Menu

WIP v1.0.0 Updated 2026-07-07

A flat dropdown list of options or actions anchored to a trigger. It is the shared list behind the select and time-picker popups, and — with the .ds-menu-anchor wrapper — a standalone action menu (kebab / "Actions" button) for the datatable and toolbars. It replaces the legacy BtnMenu, adding full keyboard support, per-item disabled, leading icons and a divider. Flat surface, separated by a border — no elevation.

1Anatomy

ℹ️
Source Extracted from input.css (the select / time-picker dropdown list) and merged with the standalone anchoring from the office-migration overlays workbench (legacy component: BtnMenu). Class contract: .ds-menu-anchor > [trigger] + .ds-menu > .ds-menu__item (.ds-menu__icon) / .ds-menu__divider.

A static menu shown open. Rows carry role="menuitem", a leading .ds-menu__icon, and an optional .ds-menu__divider. The disabled row is aria-disabled — focusable but inert.

4Usage Guidelines

✓ Do

  • Use it as the option list for a select / combobox, or as a standalone action menu on a kebab / "Actions" button
  • Give every trigger aria-haspopup and keep aria-expanded in sync
  • Group destructive or unrelated actions below a .ds-menu__divider
  • Keep the disabled item in the list (with aria-disabled) so it stays discoverable, rather than hiding it
  • Swap to a ds-sheet list on mobile — a floating menu is a poor touch target

✗ Don't

  • Re-declare .ds-menu rules on a page or fork them into another stylesheet — link menu.css
  • Nest interactive controls other than the row button inside an item (except the multi-select checkbox)
  • Use it for a handful of always-visible choices — that's a radio group or segmented control
  • Rely on hover alone to reveal or operate it — every path must work by keyboard
  • Add elevation / shadow — the menu is flat, separated by its border (matches the select and FAB)

Under @media (prefers-reduced-motion: reduce) the pop-in animation and the item background transition are dropped — the menu appears instantly and rows change colour without a fade. The select / time-picker reveal is a plain display toggle with no animation.

8Engineering

The list and item styles were extracted verbatim from input.css; the standalone anchoring came from the office-migration overlays workbench. The shared CSS owns presentation and the .is-open / .ds-menu--static hooks only — the reveal logic (open/close, roving keyboard focus, typeahead, Escape-returns-focus, outside-click, mobile sheet swap) is behaviour the consuming platform implements. This component replaces the legacy BtnMenu.

Acceptance criteria

  • List: flat surface (colour/surface/default), 1px colour/border/default, radius/m, no shadow, scrolls past 280px
  • Trigger carries aria-haspopup="menu" and a synced aria-expanded
  • Opens on click / Enter / Space / ArrowDown (first) / ArrowUp (last); focus moves into the list
  • Arrows wrap; Home / End jump; typeahead focuses the first prefix match
  • Disabled rows are reachable by arrows but never activate (aria-disabled)
  • Escape closes and returns focus to the trigger; Tab and outside click also close
  • Standalone rows are ≥44px; roving focus draws the colour/border/focus ring
  • Below the mobile breakpoint the same actions present as a ds-sheet list
  • prefers-reduced-motion: pop-in and item transition removed

9Open Questions
