Component

Time Picker

WIP v1.0.0 Updated 2026-07-07

A keyboard-first segmented hh:mm[:ss] + AM/PM field that lives inside the shipped ds-input box. Each segment is an overtype cell: type digits (auto-advancing), ArrowUp/Down to step, A/P to set the meridiem — or open a tap-assist list of common times. Unlike the legacy InputTimePicker, it does not auto-prefill the current time by default; prefill is an explicit opt-in. Replaces legacy InputTimePicker.

4Usage Guidelines

⚠️
No auto-prefill by default — a deliberate reversal of the legacy InputTimePicker. The legacy field silently filled in the current time the moment it received focus, which surprised users and led to accidentally-saved wrong times (they never meant to set a time at all). Stardust starts empty: a time picker with no value is valid until a form requires one. Prefill is available as an explicit opt-in (prefillOnFocus) for the rare flow that genuinely wants "now" as the default — use it consciously, not as a habit.

✓ Do

  • Use it wherever a single clock time is entered — pick-up, drop-off, start/end of a session
  • Let people type: overtype digits auto-advance, so 9 3 0 lands on 09:30 fast
  • Offer the tap-assist list for touch and for common round times
  • Enable showSeconds only when seconds are genuinely meaningful (e.g. logging)
  • Leave the field empty by default and let the form decide whether a time is required

✗ Don't

  • Auto-prefill the current time out of habit — it's opt-in for a reason (see above)
  • Use it to pick a duration — that's a different input, not a clock time
  • Use it for a date — pair it with the Date Picker instead
  • Reimplement the segments as one free-text field — you lose the overtype, stepping and validation
  • Show seconds "just in case" — the extra segment is friction when it isn't needed

Acceptance criteria

  • Segments overtype and auto-advance; ArrowUp/Down step with correct wrap; A/P set the meridiem
  • Single digits normalise on blur (9 → 09)
  • Impossible (hours > 12, minutes/seconds > 59) or incomplete times show the ds-input error state on blur; empty is valid
  • The clock toggle opens a ds-menu listbox at 15- or 30-minute granularity; picking fills the segments and returns focus to the toggle
  • showSeconds adds the ss segment + separator; toggling it resets the field
  • No auto-prefill by default; prefillOnFocus is opt-in and fills only an empty field on first focus
  • Each segment ≥44px tall; meridiem and toggle ≥44px; segment font-size 16px
  • prefers-reduced-motion: segment/toggle/meridiem transitions removed

9Open Questions

ℹ️
24-hour mode & locale. v1.0 is a 12-hour clock with an explicit AM/PM meridiem. A 24-hour variant (hours 00–23, no meridiem) and locale-driven default formatting are deferred — the segmented structure supports both without changing the class contract. Whether the tap-assist list should respect a business's open hours (rather than the full 24h) is also open.
