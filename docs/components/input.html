Component

Input

WIP — Phase 1 v1.0.0 Updated 2026-06-11 DRI: Mason

A unified input shell covering text, password, search, send, and textarea variants (Phase 1). Select, combobox, date, time, and file variants follow in later phases. All variants share identical visual states, tokens, and label behaviour.

4Usage Guidelines

Label placement — outside vs inside

✓ Do — outside label
Label always visible. Passes WCAG SC 1.3.5 (Input Purpose). Works with autofill and screen readers without special handling.
✗ Don't — inside label (new work)
Label disappears once user types, making the field's purpose ambiguous. Fails SC 1.3.5 for long-form inputs.
Service name

Use this component when

  • Collecting user text input in a form
  • Providing a search interface (type=search)
  • Building a chat-style send interaction (type=send)
  • Multi-line free text (type=textarea)

Don't use this component when

  • Choosing from a fixed list — use DsInputSelect (Phase 2)
  • Picking a date or time — use DsInputDate / DsInputTime (Phase 4)
  • Toggle a boolean setting — use Toggle

Helper text

  • Use for format hints, constraints, or supplemental guidance ("dd/mm/yyyy", "Min 8 characters")
  • Error message replaces helper text — don't show both simultaneously
  • Keep it to one line; wrap if necessary but avoid multi-paragraph

Placeholder vs label

  • Placeholder is supplemental — it disappears when the user types. Never use it as the sole label.
  • If you have both a label and a placeholder, the placeholder should show an example value ("E.g. Ada Lovelace"), not repeat the label

6Accessibility

🏷️
Label association
Always pair <label for="…"> with <input id="…"> or wrap the input in the <label>. Never use placeholder as the sole label — it is not announced as the accessible name by all screen readers, and disappears when the user types.
📐
Touch target
Default height 56px exceeds iOS 44pt and Android 48dp minimums. Comfortable (48px) meets the minimum. Compact (40px) is below the Android recommendation — use sparingly. Clear button is 24×24 visual but wraps in a 44×44 tappable area in native implementations.
🎨
Colour contrast
Input text (text/primary #252525 on white): 16.1:1 — AAA.
Placeholder (text/secondary #838383 on white): 4.48:1 — meets AA for large text; borderline for 16px normal weight. Placeholder is supplemental only — never the sole label.
Error border (#D21D3E on white): passes non-text contrast 3:1 (WCAG 1.4.11).
Focus ring (#DFF2F1 on white): supplemental — the border colour change carries the primary error/focus signal for users who can't perceive colour alone.
⌨️
Keyboard
Tab moves focus into the field. Clear button and action button are in the natural tab order. Search: Enter in the field fires the @search event. Send: Enter fires @send when the field is non-empty. Password eye toggle: Space / Enter toggles visibility; aria-label updates to "Show password" / "Hide password".
❗
Error announcements
Connect the helper/error text to the input via aria-describedby. When the error state activates, set aria-invalid="true" on the input. The helper text element should have a stable ID so screen readers can follow the association.
🔊
Screen reader
Web Native <input> / <textarea> — all roles and states built-in.
iOS Use UITextField / UITextView with accessibilityLabel set from the outside label prop.
Android Use TextInputLayout / EditText with contentDescription and hint set appropriately.
⚠️
Inside label fails SC 1.3.5 The inside label (floating-label pattern) violates WCAG 2.1 Success Criterion 1.3.5 (Identify Input Purpose) in many implementations because the label is replaced by the value once filled, making it impossible for assistive technology to identify the purpose from a visible label. This is one of the primary reasons it is deprecated.

8Engineering

ℹ️
Architecture note DsInput is the base shell. Each named variant (DsInputSearch, DsInputSend, DsInputTextarea etc.) is a thin wrapper that passes a type prop and wires up the appropriate event/button logic. Phase 2 adds DsInputSelect and DsInputCombobox on top of the same shell.

Props interface — Phase 1

v1.0.0
TypeScript 
type InputType     = 'text' | 'email' | 'password' | 'number' | 'url'
                   | 'textarea' | 'search' | 'send';
type InputDensity  = 'default' | 'comfortable' | 'compact';

interface DsInputProps {
  // Value
  modelValue?:        string;
  type?:              InputType;    // default: 'text'

  // Labels
  label?:             string;       // outside label — preferred
  /** @deprecated Use label instead. Emits console.warn in dev. */
  insideLabel?:       string;
  placeholder?:       string;
  helperText?:        string;

  // State
  error?:             boolean;
  errorMessage?:      string;       // overrides helperText when error=true
  disabled?:          boolean;
  readonly?:          boolean;
  required?:          boolean;

  // Icons
  leadingIcon?:       string;       // icon name from ds-icon
  trailingIcon?:      string;       // icon name from ds-icon
  clearable?:         boolean;

  // Layout
  density?:           InputDensity; // default: 'default'

  // Accessibility
  id?:                string;       // forwarded to native <input>
  name?:              string;
  autocomplete?:      string;
  ariaLabel?:         string;       // for icon-only or labelless fields
  ariaDescribedby?:   string;       // supplement to auto-wired helperText
}

// Events
// @update:modelValue(value: string)
// @search(term: string)         — InputSearch: Enter or search button
// @send(message: string)        — InputSend: Enter or send button
// @clear()                      — clearable=true: X button clicked
// @focus(event: FocusEvent)
// @blur(event: FocusEvent)
// @change(event: Event)

Acceptance criteria

  • All 5 states render correctly: unfilled, focus, filled, error, disabled
  • 3 densities: 56px, 48px, 40px
  • Focus ring appears on :focus-within only — never on hover
  • Outside label: always visible, properly associated via for/id
  • Inside label: renders but emits console.warn in dev builds
  • Error state: sets aria-invalid="true" and aria-describedby on helper text
  • Clear button: hidden when empty; visible when filled; fires @clear and clears model value
  • Password: eye button toggles type; aria-label updates to match current state
  • Search: Enter and search button both fire @search with current value
  • Send: send button disabled when empty; Enter and button fire @send
  • Textarea: height auto, min-height --input-height, resizable vertically
  • 16px font-size on <input> prevents iOS auto-zoom
  • All token values match Section 5 tables

9Open Questions

⚠️
1 · Inside label sunset timeline The insideLabel prop is marked deprecated in this spec. A 6-month removal window is recommended (i.e., remove in the next major version after all PHP/Vue3 usages have been migrated to outside label). Confirm the timeline with the PHP team before setting a hard removal date.
⚠️
2 · Figma gap — mobile bottom sheet overlays not yet designed Phase 3 (mobile overlays for Select/Combobox, Date, Time) requires Figma frames for bottom sheet variants before implementation can begin. These should be designed and reviewed before Phase 3 kicks off.
ℹ️
3 · Compact density below Android touch minimum Compact (40px) is 4dp below the Android 48dp minimum touch target recommendation. Consider whether compact density should be restricted to non-interactive contexts (e.g. read-only / display-only fields) or supplemented with invisible padding.
ℹ️
4 · Filled state border — no Figma differentiation from Unfilled The Figma component shows the same border/strong (#838383) for both unfilled and filled states. The only visual change when filled is input text colour switching from placeholder grey to text/primary. This matches the Figma spec but differs from some other design systems. Confirm this is intentional.
ℹ️
5 · Required field indicator No asterisk or "Required" label is defined in the Figma component. Confirm whether required fields should show an asterisk (*) appended to the label, a "(required)" text, or remain unmarked in favour of marking optional fields instead.
