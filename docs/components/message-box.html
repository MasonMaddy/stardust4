Component

Message Box

WIP v1.0.0 Updated 2026-06-15 DRI: Mason

An inline persistent feedback component — a full-width banner that stays visible until the user explicitly dismisses it. Five semantic colour variants convey status (success, warning, error, info, or neutral). An optional body line and optional CTA button accommodate lightweight actions without leaving the current screen. Not a toast — use ds-toast for transient auto-dismiss notifications.

1Container.ds-message-box

Root element. display: flex; flex-direction: column; align-items: flex-end — positions the optional action button right-aligned below the content row. 16px padding · 16px radius · 1px border. Full width. Required.

2Row.ds-message-box__row

Horizontal flex row holding the icon column and text column side by side. gap: 4px · align-items: flex-start so the icon aligns to the first line of text. width: 100%. Required.

3Icon.ds-message-box__icon

22×22px flex wrapper; inner SVG is 16×16px. flex-shrink: 0 · aria-hidden="true" · tinted via color: currentColor per variant. Icon is swappable — use any iconsOutline glyph that fits the message context. Required.

4Text column.ds-message-box__text

flex: 1; min-width: 0 — takes remaining width after the icon, enables text wrapping. Stacks the title row and optional body with gap: 2px. Required.

5Title row.ds-message-box__title-row

Horizontal flex row holding the title text and close button on the same baseline. align-items: center · gap: 8px. Required (even when no close button is shown, to preserve layout).

6Title.ds-message-box__title

font-size/sm · font-weight/bold · line-height/sm · colour/text/text-primary · flex: 1 to push close button to the trailing edge. Required.

7Close.ds-message-box__close

20×20px icon button · aria-label="Dismiss" · role="button" (implicit) · tinted per variant via color. Triggers mbDismiss() which adds .is-dismissed to the container. Optional — omit if the message is not dismissible.

Supporting detail text

font-size/sm · font-weight/regular · line-height/sm · colour/text/text-primary. Aligns naturally under the title with no padding tricks — the two-column layout handles it. Optional.

40px ghost button · right-aligned below the row · tinted per variant (border + text = variant accent colour). Keep labels short — 1–3 words. Optional.

4Usage guidelines

Use this component when

  • You need inline persistent feedback that the user must consciously dismiss — e.g. "Your payment failed", "Session nearly full", "Enrolment submitted".
  • The message is tied to the current page context and should stay visible while the user acts on it.
  • You need a lightweight CTA alongside the feedback (action button) without navigating away.
  • The severity warrants one of the 5 semantic colours (neutral, success, warning, error, info).

Don't use this component when

  • Transient / auto-dismiss feedback — use ds-toast instead. Message Box never auto-dismisses.
  • Field-level validation errors — use the error state on the ds-input component, with the error message directly below the field.
  • Blocking modal feedback — use a ds-dialog if the user must acknowledge before any other action is possible.
  • Promotional or marketing content — this component carries semantic urgency; misusing it for promotions erodes user trust.

Content guidelines

  • Title: 3–8 words. Sentence case. State the outcome or issue directly — no filler like "Heads up!" or "FYI".
  • Body: 1–2 sentences maximum. Explain the consequence or what the user should do next. Omit if the title is self-explanatory.
  • Action label: 1–3 words. Verb-first. Match the consequence — "Book now", "Update details", "View enrolment". Avoid "Click here" or "Learn more".
  • Icon: Swap the default icon to one that matches the message context (e.g. use a calendar icon for a booking warning, not a generic info circle).

Do / Don't

✓ Do

Payment failed

We couldn't charge the card on file.

Clear title, brief body, direct action label. Role="alert" for error severity so screen readers announce it immediately.

✕ Don't

Heads up! Something went wrong and we couldn't complete the last action you tried to do in our system

Vague filler title, no body, overly long action label, wrong role (red should use role="alert").

6Accessibility

ARIA role role="status" for grey / green / orange / purple — politely announces to screen readers when the message appears.
role="alert" for red (error) — live region with aria-live="assertive" implicit, interrupts immediately. Only use on genuinely urgent / blocking errors.
Container label aria-label="[Severity]" on the root element — e.g. aria-label="Error". Provides context before the title is read.
Icon aria-hidden="true" on the icon span — it is purely decorative. Severity is already conveyed by the container label and title text.
Close button aria-label="Dismiss" — the ×SVG carries no text, so an explicit label is required. After dismiss, focus must be managed by the calling screen (move to the next logical element).
Action button Visible label text is the accessible name — no additional aria attributes needed. Ensure the label describes the action, not just "Click here".
Keyboard Tab — cycles to close button, then action button (if present).
Enter / Space — activates the focused button.
The container itself is not focusable — only its interactive children.
Touch target Close button: 20×20px visual, 44×44pt minimum required — ensure the touch target is expanded in the native implementation (padding or separate hit area). Action button: 40px height meets the minimum.
Contrast Title / body text: #252525 on all variant backgrounds — passes WCAG AA (≥ 4.5:1) on all 5 variants.
Icon / close tint: checked per variant against the variant background — all pass AA for UI components (≥ 3:1).
Reduced motion The component has no entrance/exit animation. The close / action hover opacity transition is suppressed via @media (prefers-reduced-motion: reduce) { * { transition: none !important } } in the global stylesheet.

HTML structure

<!-- Title + body + close + action (all parts) -->
<div class="ds-message-box ds-message-box--green"
     role="status"
     aria-label="Success">
  <div class="ds-message-box__row">
    <span class="ds-message-box__icon" aria-hidden="true">
      <!-- 16×16 SVG from iconsOutline -->
    </span>
    <div class="ds-message-box__text">
      <div class="ds-message-box__title-row">
        <p class="ds-message-box__title">Enrolment submitted</p>
        <button class="ds-message-box__close"
                aria-label="Dismiss"
                onclick="mbDismiss(this)">
          <!-- 16×16 close SVG -->
        </button>
      </div>
      <p class="ds-message-box__body">
        Liam's enrolment has been sent to the centre.
      </p>
    </div>
  </div>
  <button class="ds-message-box__action">View enrolment</button>
</div>

<!-- Dismiss JS (vanilla — adapt for your framework) -->
<script>
  /* SECURITY: reads only the closest .ds-message-box ancestor,
     adds only a CSS class. No innerHTML, no user content rendered. */
  function mbDismiss(btn) {
    var box = btn.closest('.ds-message-box');
    if (box) box.classList.add('is-dismissed');
  }
</script>
