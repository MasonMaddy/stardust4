Component

Button

WIP v1.0.0 Last updated 2026-06-02 DRI: Mason

Buttons trigger actions. They signal what happens next, communicate priority, and guide users through tasks. Stardust offers three variants — Solid, Ghost, and Minimal — across Text and FAB (floating action button) styles.

4Usage Guidelines

Use this component when

  • Triggering an immediate action (Submit, Save, Delete, Create)
  • Navigating to a new context as a CTA (e.g. "Get started")
  • Confirming or dismissing a modal/dialog
  • Primary action on a card or list row

Don't use this component when

  • Inline navigation only — use a text link (<a>) instead.
  • Secondary filter actions — use a Chip or Toggle.
  • More than 2 solid buttons in one view — one primary action at a time.
  • Destructive actions without a confirmation — pair with a dialog.

Content guidelines

  • Label: verb + noun where space allows (e.g. "Save changes", "Add member")
  • Sentence case only — not ALL CAPS, not Title Case
  • Max ~4 words. If longer, rethink the action.
  • Avoid generic labels ("Click here", "Submit") — be specific about the outcome

Layout guidelines

  • Minimum touch target: 44×44pt on mobile (md size satisfies this)
  • Group at most 2 buttons side-by-side; left = primary, right = secondary/ghost
  • Full-width buttons on mobile viewports are acceptable for primary CTAs
  • Gap between paired buttons: spacing/stack-gap/default (8px)

Do / Don't

✓ Do
Use one Solid button per view for the primary action. Pair with a Ghost or Minimal for secondary actions.

✗ Don't
Don't stack two Solid buttons — it dilutes hierarchy and confuses priority.

✓ Do
Use a specific, outcome-oriented label.

✗ Don't
Don't use vague, non-specific labels that don't tell the user what happens.

6Accessibility

📐
Touch target
Minimum 44×44pt on mobile. The md size (8px top + 8px bottom padding + ~23px line-height = ~39px) must be supplemented with a transparent tap-target overlay or increased padding on native. The lg size (12px + ~24px + 12px = 48px) comfortably meets the 44pt minimum.
🎨
Contrast ratios
Solid default: White (#FFFFFF) on Teal (#00776B) = 4.65:1 ✓ AA text
Solid hover: White on #008480 = 4.2:1 ✓ AA text (borderline — verify with tool)
Ghost label: Teal (#00776B) on White = 4.65:1 ✓ AA
Disabled: #BDBDBD on #F6F6F6 = ~1.6:1 — intentionally below AA (disabled state exemption per WCAG 1.4.3)
🔊
Screen reader
Role: button (native <button> element preferred over role="button").
Accessible name: from visible label text. Icon-only buttons must include aria-label.
Disabled: use disabled attribute (not aria-disabled alone) so the button is removed from tab order.
⌨️
Keyboard
Tab — moves focus to button
Space / Enter — activates button
Disabled buttons must not receive focus (tabindex="-1" or disabled attribute)
🎬
Motion / reduced-motion
This component animates background-color transitions (150ms) and press scale (0.98). When prefers-reduced-motion: reduce is set, all transitions must be disabled. See Section 7.
📱
Platform notes
iOS VoiceOver: Native UIButton or accessibilityTraits = .button. Double-tap to activate.
Android TalkBack: Set contentDescription for icon-only variants. Default label from button text is read automatically on React Native's Pressable/TouchableOpacity.

Acceptance criteria

Pre-filled by design
  • All 3 type variants (Solid, Ghost, Minimal) render correctly
  • FAB style uses border-radius: 9999px
  • All 4 states (default, hover, pressed, disabled) are visually distinct
  • Focus ring appears on keyboard navigation — not on mouse click
  • Disabled button is not focusable via keyboard
  • Touch target meets 44×44pt minimum on mobile (use lg size on native)
  • All token values match the Token References table (Section 5)
  • prefers-reduced-motion disables transitions
  • Icon-only FAB includes aria-label or accessibilityLabel
