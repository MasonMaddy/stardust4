Button

WIP v1.0.0 Updated 2026-06-02 DRI: Mason

Buttons trigger actions. Stardust offers three variants — Solid, Ghost, and Minimal — across Text and FAB (floating action button) styles, with consistent states across web, iOS, and Android.

4Usage Guidelines

Use this component when

  • Triggering an immediate action (Submit, Save, Create)
  • Navigating to a new context as a primary CTA
  • Confirming or dismissing a modal/dialog

Don't use this component when

  • Inline navigation only — use a text link instead
  • More than 2 Solid buttons in one view — one primary action at a time
  • Destructive actions without confirmation — pair with a dialog

Content guidelines

  • Label: verb + noun where space allows ("Save changes", "Add member")
  • Sentence case — not ALL CAPS or Title Case
  • Max ~4 words. If longer, reconsider the action scope.

Do / Don't

✓ Do
One Solid for primary, Ghost/Minimal for secondary.
✗ Don't
Two Solid buttons dilute hierarchy.

6Accessibility

📐
Touch target
Minimum 44×44pt (all platforms). The md size (8+23+8 = ~39px height) should use extra tap-area padding on native. lg (12+24+12 = 48px) meets 44pt comfortably.
🎨
Contrast ratios
Solid default: White on #00776B = 4.65:1 ✓ WCAG AA
Solid hover: White on #008480 = 4.2:1 ✓ WCAG AA
Ghost label: #00776B on White = 4.65:1 ✓ WCAG AA
Disabled: #BDBDBD on #F6F6F6 ≈ 1.6:1 — intentional (WCAG 1.4.3 exempts disabled states)
⌨️
Keyboard (web)
Tab — focus · Space/Enter — activate · Disabled buttons must not receive focus
🔊
Screen reader
Web Use native <button> element. Icon-only FAB requires aria-label.
iOS VoiceOver reads button label automatically. Double-tap to activate. Add accessibilityLabel for icon-only FAB.
Android TalkBack reads button text automatically. Add contentDescription for icon-only FAB.
🎬
Motion
Web: wrap transitions in @media (prefers-reduced-motion: no-preference). Native: respect system accessibility settings for reduced motion.

Acceptance criteria

Pre-filled by design
  • All 3 types (Solid, Ghost, Minimal) render correctly on all target platforms
  • FAB style uses full border radius
  • All states (default, hover, pressed, disabled) are visually distinct
  • Focus ring appears on keyboard/switch-access navigation — not on tap/click
  • Disabled button is not interactive and not reachable by keyboard
  • Touch target ≥ 44×44pt (iOS) / 48×48dp (Android)
  • All token values match the Token References table (Section 5)
  • Web: prefers-reduced-motion disables transitions
  • Native: respects system reduced-motion accessibility setting
  • Icon-only FAB has accessibilityLabel / accessibilityHint on all platforms
