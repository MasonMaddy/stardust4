Component

Button

Published v1.1.0 Updated 2026-06-02 DRI: Mason

Buttons trigger immediate actions. Four types — Solid, Ghost, Minimal, and Destructive — share a fixed 48px height, optional leading/trailing icons, and hug or full-width layout. Icon-only and FAB actions belong to the separate Icon Button component.

4Usage Guidelines

Use this component when

  • Triggering an immediate action (Submit, Save, Create, Delete)
  • Primary or secondary CTA on a form, modal, or card
  • Confirming or dismissing a dialog

Don't use this component when

  • Inline navigation — use a text link
  • Filter or toggle actions — use Chip or Toggle
  • More than one Solid button in a view
  • Destructive actions without confirmation
  • Icon-only or FAB actions — use Icon Button
  • Submission in progress — disable and overlay a loading animation

Content

  • Sentence case labels ("Save changes")
  • Verb + noun where space allows
  • Max ~4 words

Do / Don't

✓ Do
One Solid for the primary action; Ghost or Minimal for secondary.
✗ Don't
Two Solid buttons compete for attention and dilute hierarchy.

6Accessibility

📐
Touch target
Fixed 48×48px height meets iOS (44pt) and Android (48dp) minimums on all types including Minimal.
🎨
Contrast
Solid default: white on action/primary = 4.65:1 (AA).
Ghost / Minimal label: action/primary on white = 4.65:1 (AA).
Disabled states are intentionally low contrast — WCAG 1.4.3 exempts disabled controls.
⌨️
Keyboard
Tab to focus · Space / Enter to activate. Focus glow on :focus-visible only.
🔊
Screen reader
Web Native <button>.
iOS VoiceOver reads label; set accessibilityLabel when text is hidden.
Android TalkBack reads label; set contentDescription when text is hidden.

Acceptance criteria

  • All 4 types render on web, iOS, and Android
  • 5 states per type via pseudo-states + isDisabled
  • Focus glow on keyboard focus only — not hover or click
  • 48px height, radius/lg, icons 16px and off by default
  • Ghost: 1px border default, 2px on focus
  • Destructive: full state set including disabled with text/disabled fill
  • No loading variant — loading animation pattern used instead
  • Icon-only / FAB → Icon Button component
  • All token values match Section 5 tables

9Open Questions

Intentional design decisions worth confirming before engineering finalises tokens and behaviour. Not blockers for the current spec — flag in design review.

⚠️
1 · Minimal focus has no outer glow Unlike Solid, Ghost, and Destructive, the Minimal focus state carries no box-shadow glow — only a 1px action/pressed border. Confirm this is deliberate for the lighter visual weight before finalising the implementation.
⚠️
2 · Destructive disabled uses text/text-disabled fill Destructive disabled background is colour/text/text-disabled (#BDBDBD), not colour/action/disabled. This differs from Solid disabled. Confirm before engineering tokens are finalised.
ℹ️
3 · Hover variants in Figma may carry focus shadow layers Figma static hover frames may include a shadow layer that visually resembles the focus glow. Implementation must apply the glow on :focus-visible only — never on :hover.
ℹ️
4 · colour/surface/cyan and colour/feedback/success/subtle share a hex Minimal hover uses colour/surface/cyan (#DFF2F1); Ghost hover/focus/pressed uses colour/feedback/success/subtle (#DFF2F1). They resolve to the same value — consider aliasing one to the other in the token library to remove the duplication.
ℹ️
5 · Single typography size — no sm / lg tier Button v1.1 has one size (48px height, 16px label). If density variants are needed, add a new size axis in a future version rather than retrofitting the existing component set.
⚠️
6 · Icon Button component not yet specced Usage guidelines (and acceptance criteria) defer icon-only and FAB actions to an Icon Button component. That component set does not yet have a published spec. Needed before icon-only actions can be implemented.
