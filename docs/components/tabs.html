Component

Tabs

WIP v1.0.0 Updated 2026-07-07

A single-select tablist for switching between peer sections of one view. A roving-tabindex strip with an animated active indicator, start / center / end alignment, and — the part the legacy NavTabs never had — a first-class overflow story: --scrollable turns the strip into a scroll-snap track with gradient edge fades and auto-scroll-to-active, so 10+ tabs work by touch, trackpad, and keyboard instead of squashing.

Scrollable overflow — the story NavTabs never had

4Usage Guidelines

✓ Do

  • Use to switch between peer sections of a single view or record
  • Keep labels to a short noun phrase — a word or two, in sentence case
  • Reach for --scrollable the moment tabs can outgrow their container (5–10+ sections)
  • Keep the default (first-enabled) tab meaningful — it's what users land on
  • Pair each tab with exactly one panel and wire aria-controls/aria-labelledby

✗ Don't

  • Use tabs for a sequence or a wizard — that's a stepper, not peer sections
  • Use tabs to navigate between separate pages — that's site nav
  • Let a fixed strip squash labels — switch to --scrollable instead
  • Nest a second row of tabs inside a panel — reconsider the information architecture
  • Disable so many tabs that the strip mostly can't be used — hide them instead

Under @media (prefers-reduced-motion: reduce) the tab, indicator, and fade transitions are removed and the indicator jumps to the selected tab. On first paint the indicator is placed without animating from width 0, so it never streaks in from the left.

8Engineering

Extracted from the office-migration structure workbench. The shared CSS owns presentation, the --active/--disabled/--fade-* hooks, and the indicator's transition; the consuming platform owns the behaviour — roving tabindex, Arrow/Home/End nav, indicator measurement (set width/translateX from the active tab), fade toggling from scroll position, and auto-scroll-into-view.

Acceptance criteria

  • Roles wired: tablist / tab / tabpanel with aria-selected, aria-controls, aria-labelledby
  • Roving tabindex: only the active tab is a Tab stop; Arrow ←/→ wrap and skip disabled; Home/End jump
  • Automatic activation — focusing a tab selects it and shows its panel
  • Indicator slides to the active tab (200ms) and is placed without animating on first paint
  • Active-label bold is width-reserved so switching never jitters the strip
  • Alignment start/center/end apply when tabs fit their container
  • scrollable: scroll-snap track, hidden scrollbar, edge fades toggled from scroll position, active tab auto-scrolled into view
  • Disabled tabs use aria-disabled, are unselectable, and are skipped by Arrow-key nav
  • Focus ring is an inset 2px colour/border/focus ring via :focus-visible
  • prefers-reduced-motion: indicator/hover/fade transitions removed; auto-scroll jumps instead of animating

9Open Questions

ℹ️
Manual activation, icons, and vertical tabs v1.0 is horizontal, text-only, automatic-activation tabs. Manual activation (Enter/Space to select, for heavy panels), leading icons in tab labels, and a vertical orientation are deferred. Each can be added without breaking the existing contract if a future surface needs it.
