Handoff Standard

What a complete design-to-engineering handoff contains, and the building blocks Stardust already provides for it.

What a handoff package contains

  • A runnable HTML prototype built against real DS components and tokens.
  • A spec document — annotated states, token references for every decision, component list with DS links, interaction notes, and platform differences.
  • Figma source links to the originating frames.
  • The component manifest — machine-readable references engineering (or an agent) can bootstrap from.

The component manifest

Stardust publishes a machine-readable manifest for every component, so handoff doesn't depend on reading rendered pages:

ArtifactUse
/llms.txtFlat entry point — point an agent here first.
/api/index.jsonCatalogue of every component.
Component APIThe human guide to the manifests (classes, variants, tokens, props).
DTCG tokensThe full token set for any platform.

Per-platform code

Every component page carries platform implementation guidance in its Engineering section (Vue 3, SwiftUI, Jetpack Compose) — e.g. Button → Engineering. The Getting Started guides collect the per-platform setup steps and link into these sections.
