How we take a component or screen from idea to a tested implementation — and the planned design-to-handoff pipeline that builds on it.

The component workshop

New components move through a four-phase pipeline, encoded as Claude Code skills:

PhaseWhat happens
1 · ReviewAudit the Figma component against references (Material 3, iOS HIG); gap analysis + decision log.
2 · SpecExtract/produce the component spec.
3 · SandboxIterate the implementation in docs/sandbox/, testing every state beside the live library.
3.5 · GateIndependent senior-engineer + designer review of the WIP (tokens, architecture, a11y, design) before build.
4 · BuildExtract approved CSS to the shared stylesheet and produce the doc page.

Prototyping in the sandbox

The sandbox is where WIP work lives before it's a published component. It wires real --sd-* tokens and real components — not static mocks — so a prototype looks and behaves like production.

A worked example: docs/sandbox/playground-signin/ explores five sign-in directions in a device frame, with a bare capture mode for exporting screens. It's the template for the planned design-to-handoff flow below.

Planned — Figma frame → prototype → spec → handoff

🚧
Not yet built. The end-to-end flow below is the agreed design; it's documented here to capture intent. Building it (likely as a skill) is a later phase.
  1. Input — a Figma frame (via the Figma MCP) or a written prompt.
  2. Context extraction — read layout, component references, token values, and spacing; Code Connect links Figma components to their DS equivalents.
  3. HTML prototype — built in the repo against real DS components and tokens.
  4. Iteration loop — run in the browser, request changes, re-run; diffs between iterations, no context switching.
  5. Spec generation — annotated state screenshots, token references, component list with DS links, interaction notes, platform differences, Figma links.
  6. Native scaffold — correct component structure + token references + navigation logic for Swift/Jetpack engineers to refine.
  7. Handoff package — see the Handoff Standard.
