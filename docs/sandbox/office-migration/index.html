Sandbox

Office Migration Workbench

This board maps the legacy Office Vue 3 shared component library onto Stardust, so the Office/PHP team can see at a glance what is ready to use today and what is still to come. Every legacy component is accounted for with one of four statuses: Built — use today · Partial — built, uplift pending · Missing — to be designed + built here · Out — not ported.

It is now an interactive workbench: every built component below is the real thing — type in it, open it, toggle it, switch its variants — for review and tweaking here in the sandbox before anything is promoted to the library.

Goal: best-in-class UX, accessibility and responsiveness — similar to the old library where necessary, better where possible.

Data — the datatable programme

The legacy Datatable/Spreadsheet family (sticky header with search + dynamic filter chips + column editing, server/local sorting, selection, context menus, sticky pagination footer) is the deepest custom layer in the old library. Rebuilt here in phases: foundations → toolbar & filter chips → responsive card-collapse. Spreadsheet inline editing becomes a documented pattern (decision); right-click context menus become visible row actions.

Calendar — design exploration

The legacy shipped two calendars on two different foundations (Vue Cal and a modified Vuetify calendar with no documentation). Stardust replaces both with one ds-calendar — explored below in visual directions for review, with the phone presentation collapsing the month grid to an agenda list.
