Sandbox

Office Migration Board

This board maps the legacy Office Vue 3 shared component library onto Stardust, so the Office/PHP team can see at a glance what is ready to use today and what is still to come. Every legacy component is accounted for with one of four statuses: Built — use today · Partial — built, uplift pending · Missing — to be designed + built here · Out — not ported.

Goal: best-in-class UX, accessibility and responsiveness — similar to the old library where necessary, better where possible.

6 entries — 4 built · 2 partial

BtnPrimary BtnSecondary BtnSecondarySolid BtnSuccess BtnTextPrimary → ds-btn

Built

The variant matrix (solid / ghost / minimal / destructive) replaces all five legacy buttons. No dedicated success button — a deliberate decision. The legacy library's own docs carried a TODO to “refactor the buttons”; this is that refactor.

BtnIcon → ds-btn icon-only / ds-fab

Built

Icon-only buttons use the Button’s icon-only modifier; floating/prominent icon actions use the FAB.

BtnToggleGroup BtnToggle → ds-selection-pill

Partial

Same job — single/multi toggle selection. A button-styled density variant is under review.

BtnMenu → ds-menu (standalone)

Partial

.ds-menu already ships inside input.css. Extraction to a standalone component + full keyboard navigation is pending.

Avatar → ds-avatar

Built

Image, initials and icon types with round/square shapes; initials are the built-in image fallback.
