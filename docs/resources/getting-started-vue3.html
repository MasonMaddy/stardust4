Getting Started — Vue 3

Published

Install, import, and start using Stardust components in a Vue 3 project.

1. Install

Choose the package that fits your project:

Standard Vue 3 project

Auto-generated Vue 3 proxies with full TypeScript types and v-model support.

bash 
npm install @xplor-education/stardust-vue

Migrating from Vuetify

Office teams

Drop-in replacements that accept Vuetify prop patterns — :rules, color, and legacy props like dense and outlined.

bash 
npm install @xplor-education/stardust-vue-compat

See Vuetify migration below for the full prop mapping.

2. Use a component

Import and use components like any Vue component:

vue 
<script setup>
import { SdButton, SdInput } from '@xplor-education/stardust-vue';
import { ref } from 'vue';

const name = ref('');
</script>

<template>
  <SdInput v-model="name" label="Child name" placeholder="Enter name..." />
  <SdButton type="solid" @sd-click="submit">Save</SdButton>
</template>
i
v-model works out of the box SdInput binds to the value prop and listens to the sdChange event automatically.

3. Use tokens in your own styles

Stardust tokens are CSS custom properties on :root. They load globally once any component renders. Reference them in your own CSS for consistent spacing, colour, and typography:

css 
.my-card {
  padding: var(--sd-spacing-6);              /* 24px */
  border-radius: var(--sd-radius-lg);        /* 16px */
  background: var(--sd-colour-surface-default);
  border: 1px solid var(--sd-colour-border-default);
  font-family: var(--sd-font-family);        /* Inter */
  color: var(--sd-colour-text-primary);
}

See Token Architecture for the full list of --sd-* custom properties, and Component API for the machine-readable token and component export.

4. Listen to events

Components emit custom events prefixed with sd. Use the standard Vue @ shorthand:

vue 
<!-- Button -->
<SdButton @sd-click="handleSave">Save</SdButton>

<!-- Input: keystroke + change -->
<SdInput v-model="email" @sd-input="onKeystroke" @sd-change="onBlur" />

Vuetify migration

!
Office teams only This section applies to projects currently using Vuetify that want to migrate incrementally. If you are starting a new project, use @xplor-education/stardust-vue directly.

The stardust-vue-compat package provides drop-in replacements that accept Vuetify prop patterns, allowing incremental migration without rewriting templates:

vue 
<script setup>
// Replace Vuetify imports with compat imports
// import { VBtn, VTextField } from 'vuetify/components';
import { SdButton, SdInput } from '@xplor-education/stardust-vue-compat';
</script>

<template>
  <SdInput
    v-model="name"
    label="Name"
    :rules="[v => !!v || 'Required']"
    validate-on="input"
  />
  <SdButton color="primary" variant="tonal">Save</SdButton>
</template>

Prop mapping

Vuetify Stardust Notes
color="primary" type="solid" Default — no change needed
color="danger" type="destructive"
variant="outlined" type="ghost"
variant="tonal" type="ghost"
variant="text" type="minimal"
block full-width
:rules Validation via helper text Compat runs rules, surfaces errors via Stardust's error state + helper text
dense, fab, outlined, filled, solo Accepted, ignored Silently accepted to avoid template errors during migration
