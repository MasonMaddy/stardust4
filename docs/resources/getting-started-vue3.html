Getting Started — Vue 3

Published

Install

Choose the package that fits your project:

Standard Vue 3 projects

bash 
npm install @xplor-education/stardust-vue

Office teams migrating from Vuetify

bash 
npm install @xplor-education/stardust-vue-compat

The compat package re-exports all components with Vuetify-shaped prop surfaces — :rules, color, and accepted-but-ignored legacy props like dense and outlined. See Vuetify migration below for details.

Use a component

Import and use components like any Vue component:

vue 
<script setup>
import { SdButton, SdInput } from '@xplor-education/stardust-vue';
import { ref } from 'vue';

const name = ref('');
</script>

<template>
  <SdInput v-model="name" label="Child name" placeholder="Enter name..." />
  <SdButton type="solid" @sd-click="submit">Save</SdButton>
</template>

SdInput supports v-model out of the box — it binds to the value prop and listens to the sdChange event.

Consume tokens

Stardust tokens are CSS custom properties on :root — they are bundled with stardust-core and available globally once any component loads. Use them in your own styles for consistent spacing, colour, and typography:

css 
.my-card {
  padding: var(--sd-spacing-6);
  border-radius: var(--sd-radius-lg);
  background: var(--sd-colour-surface-default);
  border: 1px solid var(--sd-colour-border-default);
  font-family: var(--sd-font-family);
}

See the Token Architecture page for the full list of available --sd-* custom properties.

Events

Stardust components emit custom events prefixed with sd. In Vue templates, listen with the @ shorthand:

vue 
<!-- Button click -->
<SdButton @sd-click="handleSave">Save</SdButton>

<!-- Input events -->
<SdInput
  v-model="email"
  @sd-input="onKeystroke"
  @sd-change="onBlur"
/>

Vuetify migration

The stardust-vue-compat package provides drop-in replacements that accept Vuetify prop patterns. This allows incremental migration without rewriting templates:

vue 
<script setup>
// Replace Vuetify imports with compat imports
// import { VBtn, VTextField } from 'vuetify/components';
import { SdButton, SdInput } from '@xplor-education/stardust-vue-compat';
</script>

<template>
  <!-- Vuetify-style props still work -->
  <SdInput
    v-model="name"
    label="Name"
    :rules="[v => !!v || 'Required']"
    validate-on="input"
  />
  <SdButton color="primary" variant="tonal">Save</SdButton>
</template>

Prop mapping

Vuetify prop Stardust equivalent Notes
color="primary" type="solid" Default — no change needed.
color="danger" type="destructive"
variant="outlined" type="ghost"
variant="tonal" type="ghost"
variant="text" type="minimal"
block full-width
:rules Validation via helper text Compat layer runs rules and surfaces errors via Stardust's error modifier + helper text.
dense, fab, outlined, filled, solo Accepted, ignored These props are silently accepted to avoid template errors during migration.

Using with Tailwind

Stardust components use scoped CSS (not Shadow DOM), so Tailwind utility classes compose freely on the outside of any component:

vue 
<SdButton type="solid" class="mt-4 mx-auto">Save</SdButton>

<div class="flex gap-4 items-end">
  <SdInput label="Search" type="search" class="flex-1" />
  <SdButton type="solid">Go</SdButton>
</div>

The component's internal styles use --sd-* tokens and BEM classes — they will not conflict with Tailwind utilities.
