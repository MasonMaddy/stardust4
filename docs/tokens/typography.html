Typography

Font: Inter · 16 styles across 5 groups · Synced from Figma 2026-06-02

Stardust uses Inter as the single font family for all product text. Styles are grouped as Header, Body, Subtitle, Caption, and Overline — each with weight variants. iOS and Android use their native system fonts at equivalent sizes and weights — see the platform mapping section below.

Platform mapping

ℹ️
Native platforms use their system font at equivalent sizes and weights. iOS uses SF Pro and Android uses Roboto. The DS style name is the shared contract — e.g. Body Regular maps to Inter 16/400 on web, SF Pro Body on iOS, and Body Large (16sp) on Android. Weight variants (Regular / Medium / Bold) apply the same native font weight on each platform.

⚠️
iOS Dynamic Type SF Pro styles scale with the user's preferred text size setting. Always support Dynamic Type on iOS — never hardcode point sizes in UIKit/SwiftUI. Use the semantic .font() style as a starting point and let the system scale from there. Prefer .body, .headline, .caption etc. over fixed sizes.
ℹ️
Android sp units Android uses sp (scale-independent pixels) for text. 1sp = 1dp at 160dpi but scales with the user's font size preference. All DS sizes specified in px map directly to the same number in sp: Body Regular = 16sp, Caption = 12sp.
