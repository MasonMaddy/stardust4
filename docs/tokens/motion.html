ℹ️
Component-local micro-timings stay literal — for now Checkbox and Radio Button use a few component-specific values (100ms, 75ms, 120ms and the MD3 Accelerate / Decelerate curves) that intentionally remain as literal values in component CSS until the corresponding Figma variables exist. Do not "fix" them by substituting the nearest token — they are tuned per component.

Usage

⚠️
CSS-first — pending Figma variable creation These tokens exist only in tokens.css. They were extracted from values already used consistently in component CSS, not synced from Figma. Create the matching Figma variables, then fold them into the normal sync. Until then, the CSS var names are the stable contract — reference them, not the raw values.

ℹ️
Respect reduced motion Wrap non-essential animation in @media (prefers-reduced-motion: reduce) and disable or shorten it. State changes must still be perceivable without motion (WCAG 2.3.3 Animation from Interactions, AAA — treated as best practice).
